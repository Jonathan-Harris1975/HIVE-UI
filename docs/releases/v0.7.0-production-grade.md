# HIVE-UI v0.7.0: production-grade Cloudflare release

This release hardens the existing HIVE-UI application without changing the HIVE backend contract.

## Principal change

The browser no longer stores `HIVE_UI_ACCESS_KEY` or sends it with each backend request. The access key is submitted once to the same-origin Cloudflare Pages Function, which issues a signed, host-only HttpOnly session cookie.

## Compatibility

- Existing `HIVE_API_BASE_URL`, `HIVE_ADMIN_TOKEN` and `HIVE_UI_ACCESS_KEY` Cloudflare variables remain valid.
- `HIVE_UI_SESSION_TTL_SECONDS` is optional.
- No HIVE backend file or endpoint was changed.
- Existing routes, chat, files, skills and ops behaviour are preserved.

## Deployment action

Replace the HIVE-UI repository contents, commit to `main`, confirm CI passes and redeploy Cloudflare Pages. Users must sign in again because the old sessionStorage mechanism is deliberately retired.
