HIVE-UI v0.10.1 mobile contrast pass

Purpose:
- Improve dark-mode readability on mobile after UX review of HIVE screenshots.
- Remove ultra-low contrast slate-600/slate-700 text from the main UI surface.
- Lift secondary copy, placeholder copy, metadata, empty states, helper text, icon affordances and table labels to AA-safer contrast levels.

Changes:
- Promoted low-contrast text-slate-600 and text-slate-700 classes to text-slate-400.
- Promoted low-contrast text-slate-500 classes to text-slate-400 across source UI components.
- Promoted low-contrast placeholders from placeholder:text-slate-600 to placeholder:text-slate-400.
- Lifted high-value helper copy in Chat and Operations to text-slate-300 where it explains core workflow behaviour.
- Bumped UI version to 0.10.1 and build marker to mobile-contrast-pass.

UX notes:
- On the HIVE dark backgrounds, slate-600 measured roughly 2.3-2.5:1 contrast and was visually too close to the background.
- slate-500 measured roughly 3.6-4.0:1 and was still weak for small mobile text.
- slate-400 measured roughly 6.7-7.3:1 on the main HIVE backgrounds, giving safer mobile readability while preserving the dark, restrained style.
